OVERVIEW
--------
ChecklistRemote.html is a personalised run-list builder for a remote player.
It mirrors checklist.html feature-for-feature (Money, Stock, Product selector,
Collection, Notes, FAB actions) but replaces the tier-grouped layout with:

  - A FLAT LIST of businesses in a custom run order
  - An on-site BUSINESS PICKER (autocomplete search)
  - DRAG-TO-REORDER cards (HTML5 Drag & Drop)
  - RUN ORDER persisted via checklist-remote-order-config.js
  - REAL-TIME SYNC with checklist.html (Storage Event Listener)
  - COLLAPSIBLE "All Business Summary" aggregating product needs

Only OPEN businesses appear (same filter as checklist.html).
Placeholder Business Name entries are always excluded.

JS MODULES
----------

1. checklist-remote-order-config.js
   Path : c:\Users\Fire0x\Desktop\TheTycoon\js\pages\checklist-remote\checklist-remote-order-config.js
   Role : OWNS the run list and its order. All reads/writes go through here.
   Key  : localStorage key = 'checklistRemoteRunList'  (JSON array of codes)
   API  :
     window.getRemoteRunList()          -> string[]   ordered business codes
     window.saveRemoteRunList(arr)      -> void        overwrites full list
     window.addToRunList(code)          -> boolean     appends (no duplicates)
     window.removeFromRunList(code)     -> void        removes one code
     window.reorderRunList(arr)         -> void        replaces order after drag

2. checklist-remote-config.js
   Path : c:\Users\Fire0x\Desktop\TheTycoon\js\pages\checklist-remote\checklist-remote-config.js
   Role : General constants + runtime data cache. No order logic here.
   API  :
     window.allOpenBusinesses           -> object map  businessCode -> enriched object
     window.getRemoteTierColorClass(n)  -> string      CSS class for tier header

3. checklist-remote-core.js
   Path : c:\Users\Fire0x\Desktop\TheTycoon\js\pages\checklist-remote\checklist-remote-core.js
   Role : Data loading from APIs.
   Fetches (parallel):
     GET /n/businesses   -> all businesses (filters to status==='Open' only)
     GET /n/employees    -> all employees
     GET /api/n/tiers    -> all tiers (used to populate tier_name, tier_number)
   Populates window.allOpenBusinesses with enriched objects:
     {
       business_code, business_name,
       tier_id, tier_number, tier_name,   <- tier_name/tier_number added for picker search + card badge
       status, can_collect_items,
       storage_capacity, collection_storage,
       checklist_product_id,
       employees: [{ name }]
     }
   After populating, calls window.buildRemoteChecklist().

4. checklist-remote-ui.js
   Path : c:\Users\Fire0x\Desktop\TheTycoon\js\pages\checklist-remote\checklist-remote-ui.js
   Role : All rendering and interaction.
   Key functions:
     window.buildRemoteChecklist()
       Reads getRemoteRunList(), renders a card per code in order.
       Each card:
         run-order badge | drag handle | checkbox
         .business-code + .business-tier-badge ("Tier N: Name") + .business-name
         employees row
         Money / Stock / Product / Collection (if applicable) / Notes
         hide-eye button | remove button
       After render: loadRemoteProgress() -> _populateAllProductSelectors()
       Note: .business-tier-badge only rendered when biz.tier_name is truthy.

     window.initBusinessPicker()
       Wires #bizPickerInput. On input:
         Searches allOpenBusinesses by:
           - business_code  (substring, case-insensitive)
           - business_name  (substring, case-insensitive)
           - tier_number    (e.g. typing "2" matches all Tier 2 businesses)
           - tier_name      (e.g. typing "Toll" matches Toll Booth businesses)
         Shows up to 20 results. Each result shows:
           <code> - <name> . Tier N: <tier_name>
         Click -> addToRunList() -> buildRemoteChecklist() -> scroll to new card.
         Already-in-list duplicates flash the existing card instead of adding.

     window.toggleReorderMode()
       Toggles _reorderMode flag.
       When ON : adds .reorder-mode to container, enables drag events on cards.
       When OFF: removes .reorder-mode, disables drag events.
       On drop  : updates DOM order, calls reorderRunList(newOrder),
                  updates .run-order-badge numbers.

     window.calculateRemoteStockNeeded(code, maxStock)
       Mirrors checklist-calculations.js logic exactly.
       Reads .stock-target-input, writes to #stock-needed-{code}.

      window.handleRemoteProductSelection(code)
        Saves product selection to localStorage ('checklistProductTracking')
        using tiered structure: { "Tier": { "Code": "ID" } }.
        Also PATCHes /api/n/businesses/{code} with checklist_product_id.

      _restoreRemoteProductTracking()
        Restores selections from the tiered 'checklistProductTracking' key.
        Refreshed via 'storage' event for real-time sync.

      window.clearRemoteProducts()
        Resets all product selectors, clears relevant entries from
        'checklistProductTracking', and PATCHes DB.

     window.calculateAllBusinessSummary()
       Aggregates product needs across all businesses in the run list.
       Excludes checked or hidden businesses. Renders to #allBusinessSummaryBody.

     window.toggleRemoteBusinessVisibility(button)
       Hides/shows a business item (does not remove from run list).

      window.toggleFabMenu()
        Standard FAB open/close (matches checklist-ui.js).

      window.showAllRemoteBusinesses()
        Unhides all business cards in the current run list and saves progress.

      window.scrollToTop() / window.scrollToBottom()
        Utility functions to quickly navigate the list.

     window.escapeRemoteHtml(text)
       XSS helper. Defined at bottom of ui.js as window.escapeRemoteHtml.

   Product loading:
     _populateAllProductSelectors()
       Groups selectors by data-tier-id, fetches /api/n/tiers/{id}/products once
       per tier (cached in _productCache), populates <select> options.
       Then restores selections: DB first -> localStorage override.

5. checklist-remote-progress.js
   Path : c:\Users\Fire0x\Desktop\TheTycoon\js\pages\checklist-remote\checklist-remote-progress.js

6. checklist-remote-main.js
   Path : c:\Users\Fire0x\Desktop\TheTycoon\js\pages\checklist-remote\checklist-remote-main.js
   Role : Page init, DebugManager, event delegation.
   Debug: new DebugManager('checklistRemote') -> window.debug
          Prefix: [CHECKLISTREMOTE DEBUG]
          localStorage key: 'checklistRemoteDebugMode'
   DOMContentLoaded:
     initRemoteHowToUseCollapse()
     initBusinessPicker()
     updateTimes() + setInterval(updateTimes, 1000)
     loadRemoteBusinessData()
   Sync:
     window.addEventListener('storage', ...)
      Detects changes in 'businessChecklist' (progress) or
      'checklistProductTracking' (products); calls restoration logic
      and calculateAllBusinessSummary() instantly.
   Delegated events:
     input[type=checkbox] change -> saveRemoteProgress()
     .notes-input / .money-input / .collection-input / .stock-target-input input
       -> debounced (500ms) saveRemoteProgress()
     .hide-business-btn click -> toggleRemoteBusinessVisibility()
     .remove-biz-btn click    -> removeFromRunList() + buildRemoteChecklist()

================================================================================
LOCALSTORAGE KEYS
================================================================================

  checklistRemoteRunList          JSON array of business codes in run order
  businessChecklist               Progress per business (checkbox/money/stockTarget/
                                  collection/notes/hidden). SHARED with checklist.html.
  checklistProductTracking        Product selector value per business code. 
                                  Tiered structure: { "Tier Name": { "Business Code": "ID" } }.
                                  SHARED with checklist.html.
  checklistRemoteDebugMode        "true" / "false"
  checklistRemote_how_to_use_expanded  "true" / "false"

================================================================================
FEATURE DIFFERENCES vs checklist.html
================================================================================

  SAME:
    - Money, Stock (max/needed calc), Product selector, Collection, Notes
    - FAB: Check All, Uncheck All, Show All, Scroll Top, Scroll Down,
           Save Progress, Clear Products, Clear Progress, Toggle Debug
    - Time display (Brisbane + Eastern + Reboot countdown)
    - Auto-save on input (500ms debounce)
    - Hide-eye per business
    - How-to-use collapsible section
    - DebugManager integration

  DIFFERENT / NEW:
    - Flat list (no tier group headers)
    - On-site business picker with autocomplete (search by code, name,
      tier number, or tier name)
    - Tier name badge per card ("Tier N: Name" pill after business code)
    - Drag-to-reorder (Reorder button)
    - Remove-from-list button per card
    - Run-order number badge per card
    - Run list order managed by checklist-remote-order-config.js
    - REAL-TIME SYNC: Changes in one tab reflect in others instantly via storage events.
    - ALL BUSINESS SUMMARY: Aggregated product needs for the custom run list.
